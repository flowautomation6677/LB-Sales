Diretrizes de UI e Comportamento do Botão (CTA)O botão é o elemento mais importante da página. Ele não pode parecer um link de blog; deve ter a física de um software real.Estilo Base: background-color: #10B981, border-radius: 6px, color: #FFFFFF, font-weight: 700. Sem bordas (border: none).Sombra (Glow Effect): box-shadow: 0 4px 14px 0 rgba(16, 185, 129, 0.4). Cria uma aura de destaque no fundo escuro.Estado de Hover: Ao passar o mouse (desktop), o botão deve sofrer um lift (subir) usando transform: translateY(-2px) e aumentar levemente o brilho do verde (filter: brightness(1.1)), mantendo uma transição suave de 0.3s ease-in-out.Estado de Active (Clique): transform: translateY(0) e remoção sutil da sombra, simulando a pressão física.3. Animações e Interações (CSS/JS)A marca é "O Governante + O Estrategista". Animações não podem ser infantis (nada de bouncing exagerado). Devem transmitir carregamento de dados e fluidez tecnológica.Scroll Reveal (Fade Up): À medida que o usuário rola a página, os blocos de texto e imagens devem aparecer suavemente de baixo para cima (opacity: 0 para opacity: 1, com um transform: translateY(20px) para 0). Duração de 0.6s.Smooth Scroll: Ao clicar em links âncora (como no menu, se houver, ou no CTA levando ao formulário), a rolagem da página deve ser fluida (scroll-behavior: smooth no CSS root).Accordion do FAQ: Abertura e fechamento das respostas do FAQ devem expandir suavemente a altura (max-height transition) e o ícone de seta (+ para - ou chevron) deve rotacionar 180 graus.4. Diretrizes de Desenvolvimento Front-End (HTML, CSS, JS)Para garantir que a arquitetura performe bem em SEO e tenha alta conversão em dispositivos móveis (onde a maioria do tráfego corporativo rápido ocorre hoje), o desenvolvimento deve seguir estas premissas:Boas Práticas de HTML e Estrutura:Utilizar tags semânticas estritas do HTML5 (<header>, <main>, <section>, <article>, <nav>, <footer>) para garantir acessibilidade e leitura correta pelos motores de busca.Garantir a hierarquia de cabeçalhos (<h1> apenas uma vez no Hero, <h2> para blocos principais, <h3> para títulos de cards).Boas Práticas Mobile-First e CSS:Footer Centralizado: No mobile, todo o conteúdo do footer (links, logo, copyright) deve estar perfeitamente alinhado ao centro (text-align: center, com um display: flex; flex-direction: column; align-items: center;). No desktop, pode assumir um layout distribuído (space-between).Touch Targets (Área de Toque): Qualquer botão, link ou aba de FAQ deve ter uma altura mínima de 44px ou 48px para facilitar o toque de dedos no mobile, evitando cliques acidentais.Responsividade Fluida: Usar CSS clamp() para a tipografia, garantindo que os títulos reduzam de forma orgânica do Desktop (ex: 48px) para o Mobile (ex: 32px) sem quebrar palavras brutalmente.Espaçamento: Utilizar variáveis CSS (:root) para controlar paddings e margins de forma global. No mobile, as seções (<section>) devem ter um padding lateral generoso (ex: 16px ou 24px) para a interface "respirar".Performance (Core Web Vitals):Otimizar as imagens exportadas pela IA em formato .WebP para garantir carregamento ultrarrápido (impacto direto na retenção do lead).Carregar as Google Fonts de forma assíncrona ou fazer o preconnect na tag <head>.Evitar bloqueio de renderização adiando (defer) o carregamento do JavaScript não essencial para o primeiro scroll.
